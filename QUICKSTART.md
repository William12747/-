# 快速开始指南

## 安装步骤

1. **安装依赖**
   ```bash
   npm install
   ```
   
   这会安装所有必要的依赖，包括：
   - React 和相关前端库
   - Electron
   - FFmpeg 静态文件（ffmpeg-static, ffprobe-static）
   - fluent-ffmpeg

2. **启动开发模式**
   ```bash
   npm run dev
   ```
   
   这会：
   - 启动 React 开发服务器（http://localhost:3000）
   - 启动 Electron 应用窗口

## 使用流程

### 第一步：选择输入文件夹
1. 点击"输入文件夹"区域的"选择文件夹"按钮
2. 选择包含旧视频格式文件的文件夹
3. 点击"扫描视频文件"按钮
4. 等待扫描完成，系统会显示找到的视频文件数量

### 第二步：选择输出文件夹
1. 点击"输出文件夹"区域的"选择文件夹"按钮
2. 选择转换后视频的保存位置
3. 系统会自动保留原始文件夹结构

### 第三步：开始转换
1. 点击"开始转换"按钮
2. 系统会按队列顺序处理所有文件
3. 可以随时点击"暂停"或"停止"按钮
4. 查看实时进度和统计信息

### 第四步：预览和检查
1. 转换完成后，点击文件旁边的"预览"按钮
2. 在应用内预览转换后的视频
3. 如果有失败的文件，可以导出失败清单进行排查

## 支持的格式

**输入格式：**
- FLV
- ASF
- RMVB
- MPEG/MPG
- WMV
- AVI

**输出格式：**
- MP4 (H.264 + AAC)
- 兼容 QuickTime 和 Windows 媒体播放器

## 常见问题

### Q: 转换速度慢？
A: 转换速度取决于视频文件大小、分辨率和系统性能。可以暂停/继续来控制处理节奏。

### Q: 某些文件转换失败？
A: 检查失败清单中的错误信息。常见原因：
- 文件损坏
- 不支持的编码格式
- 磁盘空间不足

### Q: 如何批量处理大量文件？
A: 系统支持递归扫描子文件夹，会自动处理所有支持的视频文件。建议先测试少量文件。

## 技术说明

- 使用 FFmpeg 进行视频转换
- 输出质量：CRF 23（平衡质量和文件大小）
- 自动创建输出目录结构
- 保留原始文件夹层级关系
